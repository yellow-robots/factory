## Grading caveat

Bench grading is entirely mechanical, never an LLM judge: `tools/bench_replay.py`'s `grade()` /
`run_candidate()` seal the repo at the task's pre-solution ref, patch the held-out tests back onto the
candidate's tree from the corpus record's own stored contents — the *same PR's own* test additions,
authored by whichever AI landed the original solution, not independent ground truth — and run the target
repo's own `check_cmd`: exit 0 is `pass`, any other exit is `fail`, and an exit that means the check
harness itself couldn't even run (a missing/broken toolchain) is `ungraded-environmental`, never counted
as a graded fail.

A `pass` therefore proves only that a candidate's change makes those specific, already-anchored test
artifacts green under that repo's own check command — it is not independent proof of correctness, and it
is not graded against the original PR's approach. A candidate that solves the same task by a genuinely
correct but different path can still fail if the anchored tests happen to assert on the original
implementation's specific shape; conversely, a wrong solution that happens to satisfy the anchored
assertions would still grade `pass`. Anchored-test artifacts and a wrong-but-passing solution must stay
distinguishable at adjudication — this caveat exists so every report reader sees that distinction stated
plainly, not discovered later.

### What this column does and does not say

The grading caveat quoted above applies unchanged: held-out anchored tests measure convergence to the
merged solution under the repo's own check command, not absolute capability. Within that instrument,
on this ten-record single-repo sample, MiMo v2.5 Pro lands at the compat columns' corrected rate
(1/10) but through a different record — it clears the one-test hurdle that stopped DeepSeek, GLM, and
opus on 172-pr181, and stumbles on the one-test margin at 153-pr179 that the others clear, matching
opus's exact miss there. Its failure depths are never shallower than DeepSeek's (identical on six,
deeper on 168-pr186). At published list rates against endpoint-reported counts it prices at
**≈ $0.57 for the column** — the same order as DeepSeek's $0.62 invoice and an order of magnitude
below GLM's ≈$6–7, versus the Anthropic columns' registry-priced shadow cost (sonnet ≈$18, opus ≈$16
at `models.toml`'s `input_price_per_mtok`). Cost-per-corrected-pass lands where DeepSeek's did
(≈$0.57 for one pass), with the same fidelity conditionality — and the price-card identity with
DeepSeek (same three list rates to the cent) reads as deliberate price-matching between the two
vendors, worth knowing when a tiebreak is priced. Review-role evidence remains out of scope: this is
the build bench only.
